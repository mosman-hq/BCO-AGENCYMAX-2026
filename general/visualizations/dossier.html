Complete dossier JSON

Three top-level sections:
• primary — the host entity's complete payload (entity, golden record, source links, merge history, funding-by-year, CRA T3010 per-year, qualified donees received/given, accountability, international, related).
• merged_in_browser — array of each virtually-merged entity's complete payload (same shape as primary, one per merge).
• combined — pre-aggregated view across primary + all merges: deduplicated aliases/BNs, summed funding by year (CRA calendar + external fiscal), concatenated-with-provenance qualified donees / accountability / international data, total link count by source. Every element has a via_entity_id so you can trace each line back to the entity it came from.
